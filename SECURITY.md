# Security Policy

## Supported Scope

Security reports are welcome for:

- proof verification correctness bugs
- miner identity and signature handling bugs
- storage or artifact exposure bugs
- wallet or chain-activation safety issues
- secrets leakage in docs, scripts, or workflows

## Reporting

Please do not open public issues for suspected security problems.

Report privately to the project maintainers with:

- a short summary
- affected files or commands
- reproduction steps
- impact assessment

If the issue involves credentials or live infrastructure, rotate any affected secrets first.
